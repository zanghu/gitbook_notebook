一、物品（item）情况介绍
谛听个性化推荐中物品分为以下几种数据类型：
1.问题（英文：question, 包括工单类问题和非公单类问题）
2.博客（英文：blog）
3.想法（英文：idea）
4.资讯（英文：news）
本次提供的数据不包括资讯类数据，此类数据将在近期尽快提供。

二、用户行为介绍
谛听用户对物品可以执行的行为目前包括以下9类：
（1）赞, （2）踩, （3）关注, （4）收藏, （5）评论, （6）回复, （7）浏览, （8）作者, （9）参与
本次提供的数据都没没有包含浏览类型行为，该类型行为会在近期补充

三、用户id和物品id文件
谛听用户id和物品id放在以下四个csv文件中（位于ids_csv目录下））
1.user_ids.csv：谛听用户id
2.question_ids.csv：question类物品id
3.blog_ids.csv：blog类物品id
4.idea_ids.csv：idea类物品id
其中3类（question、blog、idea）物品的id类型为正整数，按照数字大小升序排序
用户id为32位字符串，按照字典序升序排序

四、“用户-物品”矩阵
本次提供的"用户-物品"矩阵中元素不是用户评分，而是用户对物品执行的各种类型行为的次数
本次统计的用户行为数据来自谛听过去一年的历史行为

本次提供以下三个“用户-物品”矩阵（位于u2i_csv目录下）：
user_blog.csv：用户-博客类物品
user_idea.csv：用户-想法类物品
user_question.csv：用户-问题类物品
以上三个文件中中第一行的格式是：“USER_ID,物品id_1,物品id_2,...,物品id_N”
以上三个文件中中第二行开始每一行记录一个用户对物品的历史行为, 格式是：“用户id,用户id对物品id_1的行为记录,用户id对物品id_2的行为记录,...,用户id对物品id_N的行为记录“
用户对物品的行为记录是一个长度为9的数组，数组每个元素代表用户对物品的每一类行为的次数。数组元素间用字符'|'分隔，每个元素是一个非负整数用来表示用户对某物品执行过的的该类型行为次数
长度为9的数组的每个位置数字依次对应于以下类型行为的次数：（1）赞, （2）踩, （3）关注, （4）收藏, （5）评论, （6）回复, （7）浏览, （8）作者, （9）参与

如果为用户的每个行为设置一个分值，就可以将记录用户行为“用户-物品”矩阵转化为反映用户评对物品兴趣偏好的“用户-物品”评分矩阵
